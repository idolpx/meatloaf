# Class diagrams

Mermaid class diagrams covering EVERY class in the Meatloaf source tree
(364 classes at last generation), organized by subsystem.

## Layout

- `defs/<Class>.mmd` - ONE definition per class. Two kinds:
  - hand-curated (no marker): complete, members ordered private/protected/
    public, non-obvious public members carry a ` - description` suffix.
  - generated (first line `%%gen`): parsed from the C++ header by
    `gen_defs.js`. Improve one by editing it and REMOVING the `%%gen`
    marker line - curated files are never overwritten.
- `defs/_manifest.json` - generated map: class -> header, bases, area.
- `src/<diagram>.mmd` - diagram templates (`%%include <Class>` lines plus
  layout/relations). The aspect templates are GENERATED by
  `gen_templates.js`; hand content for them lives in
  `src/extra/<area>.mmd` (appended verbatim).
- `build.js` - splices defs into templates -> top-level `.mmd` files.
- `<diagram>.mmd` - GENERATED outputs, committed so GitHub renders them.

## Building

    cd docs/classes
    node gen_defs.js       # re-parse headers; only (re)writes %%gen defs
    node gen_templates.js  # regenerate aspect templates + coverage check
    node build.js          # assemble the final diagrams
    node build.js check    # CI-style staleness check

`gen_templates.js` asserts every manifest class appears in an aspect
diagram, so new classes cannot silently go undocumented.
